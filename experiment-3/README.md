# 🧪 Experiment 3 – Multi-Page React Application using React Router

## 📌 Project Title
**StartupX – React Single Page Application (SPA)**

---

## 🎯 Objective
The objective of this experiment is to extend Experiment 2 by creating an additional webpage and implementing navigation between pages using **React Router**.  
This experiment helps in understanding **Single Page Application (SPA) architecture** and **client-side routing** in React.

---

## 🛠️ Tech Stack Used
- **React (Vite)**
- **Bootstrap & React-Bootstrap**
- **Material UI (MUI)**
- **React Router DOM**

---

---

## ✅ Features Implemented
- Component-based UI design
- Responsive layout using Bootstrap and MUI
- Multiple pages (Home & About)
- Client-side navigation using React Router
- Single Page Application behavior (no page reload)
- Clean and structured codebase

---

Installation & Setup:
Create React Project: npm create vite@latest → cd project-folder → npm install
Install UI Libraries: npm install react-bootstrap bootstrap && npm install @mui/material @emotion/react @emotion/styled
Install React Router: npm install react-router-dom
Run Project: npm run dev

Deployment:
The project is deployed using Vercel.
Example: 23BAI70472-exp3-startupx.vercel.app

Experiment Description:
Experiment 2 focused on creating a single-page React application using UI component libraries. Experiment 3 extends the same application by adding an additional page and implementing routing using React Router to enable smooth navigation between pages without reloading.
